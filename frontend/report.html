Latar Belakang

Mengapa Proyek Ini Penting?

Kendaraan listrik (EV) sedang mengalami pertumbuhan pesat di Indonesia, termasuk di Kota Medan sebagai salah satu kota terbesar di Indonesia. Namun, informasi mengenai lokasi stasiun pengisian (SPKLU) dan stasiun penukaran baterai (SPBKLU) masih tersebar dan tidak mudah diakses oleh masyarakat umum.

Proyek ini dibangun sebagai solusi berbasis Sistem Informasi Geografis (SIG) yang memanfaatkan ekstensi PostGIS pada PostgreSQL untuk menyimpan dan menganalisis data spasial berupa titik koordinat (POINT, SRID 4326/WGS84) setiap stasiun.

Dengan antarmuka peta interaktif berbasis Leaflet.js, pengguna dapat menemukan stasiun terdekat menggunakan query Nearest Neighbor, mencari stasiun dalam radius tertentu, serta melihat distribusi spasial stasiun per kecamatan di Kota Medan.

Data 54 stasiun (24 SPKLU + 30 SPBKLU) dikumpulkan dari Google Maps, situs resmi PLN, Pertamina, dan survei lapangan, kemudian disimpan dengan koordinat yang valid dan divalidasi secara spasial menggunakan PostGIS.

Metodologi

Tahapan Pengerjaan Proyek

Proyek dikerjakan secara sistematis mengikuti metodologi pengembangan SIG berbasis web modern.

Tahap 1 — Perencanaan

Analisis Kebutuhan & Desain Database

Identifikasi kebutuhan sistem, penentuan entitas (stasiun, unit pengisian, port, kabinet swap, baterai, kecamatan), normalisasi tabel, dan pemilihan tipe geometri POINT untuk stasiun dan POLYGON untuk kecamatan. Justifikasi penggunaan SRID 4326 (WGS84) untuk kompatibilitas dengan GPS dan OpenStreetMap.

Tahap 2 — Pengumpulan Data

Akuisisi Data Stasiun & Kecamatan

Pengumpulan koordinat 54 stasiun dari Google Maps dan sumber resmi. Validasi koordinat dalam batas wilayah Kota Medan. Pengumpulan data polygon 21 kecamatan. Data disimpan dalam format INSERT SQL dengan fungsi ST_SetSRID(ST_MakePoint(lon, lat), 4326).

Tahap 3 — Backend

Implementasi API Node.js + PostGIS

Pengembangan REST API dengan Express.js, koneksi ke PostgreSQL menggunakan library pg (node-postgres), implementasi 6 query PostGIS utama, autentikasi Google OAuth 2.0 + JWT, dan integrasi Cloudinary untuk penyimpanan foto stasiun.

Tahap 4 — Frontend

Pengembangan Antarmuka Web

Landing page dengan animasi AOS, custom cursor, typewriter effect, dan particle canvas. Peta interaktif Leaflet dengan marker cluster, filter, radius slider, dan nearest neighbor visualization. Dashboard untuk manajemen stasiun.

Tahap 5 — Pengujian

Testing & Optimasi Spasial Index

Pengujian semua query PostGIS menggunakan EXPLAIN ANALYZE untuk membuktikan penggunaan spatial index GIST. Pengujian endpoint API, integrasi frontend-backend, dan responsivitas antarmuka di berbagai ukuran layar.

Query PostGIS

Implementasi Query Spasial

6 query PostGIS yang diimplementasikan untuk memenuhi rubrik penilaian praktikum SIG.

1. INSERT Titik (DML Spasial)

DML
-- Menyimpan stasiun baru sebagai POINT INSERT INTO stasiun (nama, jenis, latitude, longitude, geom) VALUES ( 'SPKLU PLN Medan Baru', 'SPKLU', 3.591969, 98.666245, ST_SetSRID(ST_MakePoint(98.666245, 3.591969), 4326) );
Menyimpan koordinat stasiun sebagai tipe geometri POINT dengan SRID 4326 (WGS84). Urutan parameter: longitude dulu, baru latitude (sesuai standar GIS x,y).

2. Nearest Neighbor (KNN)

PostGIS
-- 5 stasiun terdekat dari titik user SELECT nama, jenis, ROUND(ST_Distance( geom::geography, ST_SetSRID(ST_MakePoint(98.67, 3.58), 4326)::geography )::numeric, 2) AS jarak_meter FROM stasiun ORDER BY geom <-> ST_SetSRID(ST_MakePoint(98.67, 3.58), 4326) LIMIT 5;
Operator <-> menggunakan KNN (K-Nearest Neighbor) dengan spatial index GIST untuk performa optimal. ST_Distance(::geography) untuk jarak akurat dalam meter.

3. Radius — ST_DWithin

PostGIS
-- Stasiun dalam radius 1 km dari titik user SELECT nama, jenis, alamat, ROUND(ST_Distance( geom::geography, ST_SetSRID(ST_MakePoint(98.67, 3.58), 4326)::geography )::numeric, 2) AS jarak_meter FROM stasiun WHERE ST_DWithin( geom::geography, ST_SetSRID(ST_MakePoint(98.67, 3.58), 4326)::geography, 1000 -- 1000 meter = 1 km ) ORDER BY jarak_meter;
ST_DWithin dengan konversi ::geography menggunakan satuan meter yang akurat (bukan derajat). Memanfaatkan spatial index GIST untuk query cepat.

4. Intersect — ST_Within

Analisis
-- Stasiun dalam batas kecamatan (ST_Within) SELECT s.nama, s.jenis, k.nama_kecamatan, COUNT(s.id) OVER (PARTITION BY k.id) AS total_kecamatan FROM stasiun s JOIN kecamatan k ON ST_Within(s.geom, k.geom) ORDER BY k.nama_kecamatan, s.nama; -- Agregasi per kecamatan: SELECT k.nama_kecamatan, COUNT(s.id) AS jumlah FROM kecamatan k LEFT JOIN stasiun s ON ST_Within(s.geom, k.geom) GROUP BY k.id, k.nama_kecamatan ORDER BY jumlah DESC;
ST_Within mengecek apakah titik stasiun berada di dalam polygon kecamatan. Digunakan dalam VIEW v_stasiun_per_kecamatan dan visualisasi layer kecamatan di peta.

5. Buffer — ST_Buffer

Analisis
-- Area coverage 500m sekitar stasiun id=1 SELECT nama, latitude, longitude, ST_AsGeoJSON( ST_Transform( ST_Buffer(geom::geography::geometry, 500), 4326 ) )::json AS buffer_geojson FROM stasiun WHERE id = 1; -- Hitung stasiun dalam buffer: SELECT COUNT(*) FROM stasiun WHERE ST_DWithin( geom::geography, (SELECT geom::geography FROM stasiun WHERE id=1), 500 );
Buffer dihitung dalam meter menggunakan konversi ::geography. Hasil GeoJSON ditampilkan sebagai circle di Leaflet menggunakan L.circle(). Endpoint: GET /api/report/buffer?stasiun_id=1&radius_m=500

6. Luas & EXPLAIN ANALYZE

Analisis
-- Luas kecamatan dalam km² SELECT nama_kecamatan, ROUND(ST_Area(geom::geography)::numeric / 1000000, 4) AS luas_km2 FROM kecamatan ORDER BY luas_km2 DESC; -- Buktikan penggunaan spatial index: EXPLAIN ANALYZE SELECT * FROM stasiun WHERE ST_DWithin( geom::geography, ST_SetSRID(ST_MakePoint(98.67, 3.58), 4326)::geography, 1000 ); -- Output: "Index Scan using idx_stasiun_geom"
ST_Area(::geography) menghitung luas dalam meter persegi, dibagi 1.000.000 untuk km². EXPLAIN ANALYZE membuktikan penggunaan spatial index GIST, memastikan query berjalan efisien.

Tentang Proyek

Proyek ini dibuat sebagai Tugas Akhir Praktikum Sistem Informasi Geografis (SIG) Program Studi Teknologi Rekayasa Perangkat Lunak, Politeknik Negeri Medan, Semester Genap 2024.

Proyek berfokus pada implementasi data spasial bertipe POINT untuk merepresentasikan lokasi stasiun pengisian kendaraan listrik, dengan memanfaatkan ekstensi PostGIS pada PostgreSQL untuk berbagai operasi spasial seperti query nearest neighbor, analisis radius, intersect polygon, dan kalkulasi buffer area.

Catatan Pengembangan

  • Semua query PostGIS menggunakan parameterized query ($1, $2) untuk mencegah SQL injection.
  • Konversi ::geography digunakan agar jarak dihitung dalam meter (bukan derajat), yang lebih akurat untuk wilayah non-equatorial.
  • EXPLAIN ANALYZE pada query ST_DWithin menunjukkan penggunaan "Index Scan using idx_stasiun_geom", membuktikan efektivitas GIST index.
  • Data polygon kecamatan menggunakan bounding box yang disederhanakan; untuk produksi sebaiknya menggunakan data GeoJSON dari BPS atau OpenStreetMap yang lebih akurat.
  • Backend telah dilengkapi dengan rate limiting (300 req/15 menit) dan helmet untuk keamanan HTTP headers.
